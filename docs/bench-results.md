# Betterhook benchmarks

## v1.0.1 fuzz campaign

After the v1.0.1 code-quality pass (async correctness, typed
deserialize, Cow<Hook>, cache memoization, executor refactor), the
in-process random-mutation fuzzer ran 30 s per target on the
`apps/betterhook/afl/seeds/` corpus. Zero panics across **231 million
iterations**.

| Target          | Iterations (30s) | Panics |
|-----------------|-----------------:|-------:|
| config_parse    |          634,677 |      0 |
| dag_resolver    |       17,620,596 |      0 |
| clippy_parser   |       55,518,677 |      0 |
| eslint_parser   |       86,850,458 |      0 |
| husky_importer  |       33,265,984 |      0 |
| cache_key       |       37,148,022 |      0 |
| **total**       |  **231,038,414** |  **0** |

The `eslint_parser` and `clippy_parser` numbers nearly tripled from
the v1.0.0 baseline after phase 17 (typed `Deserialize` structs
replaced `serde_json::Value` walking).

## How to reproduce

```bash
cargo build --release -p xtask
./target/release/xtask fuzz --duration 30   # ~3 min, 6 targets
./target/release/xtask fuzz --duration 120  # ~12 min for a deeper run
```

Crashing inputs land under `target/fuzz-crashes/<target>-<id>.bin`.

## Monorepo benchmark

Reproduced via `cargo xtask bench-monorepo` once `hk` and `lefthook`
are on the user's PATH. Without those tools present, the runner
reports "n/a" for the comparison rows but still measures betterhook
itself.

Generated by `cargo xtask bench-monorepo` and `cargo xtask fuzz`.
