betterhook was designed around one concrete workload: **multiple AI coding
agents running in parallel, each in its own git worktree, each committing
through a pre-commit hook.** Every feature that distinguishes it from
lefthook falls out of that constraint.

## Agent-friendly defaults

- **Soft-miss dispatch** — a worktree without a `betterhook.*` file exits 0
  silently. An agent can set up its own worktree and commit without tripping
  over a hook that doesn't apply to it yet.
- **Stable exit codes** — `0 / 1 / 2 / 3 / 4 / 5 / 64 / 124 / 130` with
  [documented meanings](/reference/exit-codes). No surprises across versions.
- **Streaming NDJSON** — every subprocess line becomes one event immediately.
  Agents don't wait for a buffered dump at the end.
- **Opt-in coordinator** — the daemon only spawns when a job declares
  `isolate`. Zero-RSS by default for agents running in ephemeral
  environments where a background daemon would be noise.

## Conductor integration

See [Conductor integration](/agents/conductor) for the specifics. The short
version: Conductor sets `BETTERHOOK_DAEMON_SOCK` in the agent's environment
so every worktree in a repo shares one pre-warmed daemon.
