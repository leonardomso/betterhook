description: "How agents should respond to a failed pre-commit hook."

The canonical flow when an agent hits a failing formatter or linter is
**run → fix → run**. betterhook is built to support this directly.

## The loop

<Steps>
  <Step title="Run the hook">
    ```sh
    betterhook run pre-commit --json
    ```
    Emits NDJSON events. The last one is `summary`.
  </Step>
  <Step title="Parse the verdict">
    If `summary.ok == true`, commit and continue.
    If `false`, collect the failing job names from `job_finished` events
    where `exit != 0`.
  </Step>
  <Step title="Check which failures are auto-fixable">
    A job is fixable if it has a `fix = ...` variant in `betterhook.toml`.
    Query it via `betterhook explain`:

    ```sh
    betterhook explain --hook pre-commit --job <name> | jq '.jobs[].fix != null'
    ```
  </Step>
  <Step title="Fix">
    ```sh
    betterhook fix --hook pre-commit --job <name>,<name2>
    ```
    Runs the `fix` variants. `stage_fixed = true` jobs automatically re-add
    their edits so the next commit sees the rewritten content.
  </Step>
  <Step title="Retry">
    ```sh
    betterhook run pre-commit --json
    ```
    If it now passes, commit. If it still fails, the remaining errors
    require real code changes — surface them to the agent's LLM for an
    actual edit.
  </Step>
</Steps>

## Example: end-to-end shell script

```sh
#!/usr/bin/env bash
set -e

# Attempt 1
output=$(betterhook run pre-commit --json)
ok=$(echo "$output" | jq -r 'select(.kind == "summary") | .ok')

if [ "$ok" = "true" ]; then
    echo "pre-commit ok on first try"
    exit 0
fi

# Collect failing jobs
failed=$(echo "$output" | jq -r 'select(.kind == "job_finished" and .exit != 0) | .job' | paste -sd, -)

if [ -z "$failed" ]; then
    echo "hook failed but no failing job reported — investigate"
    exit 1
fi

echo "jobs failed: $failed"
echo "attempting auto-fix..."

betterhook fix --hook pre-commit --job "$failed"

# Attempt 2
output=$(betterhook run pre-commit --json)
ok=$(echo "$output" | jq -r 'select(.kind == "summary") | .ok')

if [ "$ok" = "true" ]; then
    echo "pre-commit ok after fix"
    exit 0
fi

echo "pre-commit still failing after fix — real code change required"
exit 1
```

## When fix isn't enough

Some failures can't be auto-corrected:

- **Type errors** (`tsc`, `cargo check`, `mypy`)
- **Test failures** (unit tests, integration tests)
- **Security audit** findings (`cargo audit`, `npm audit`)
- **Spec / schema** mismatches

For these, the agent should surface the `job_finished.exit` code along with
the captured `line` events for that job's stderr, and feed them back into
the LLM for a real diff.

## Stage_fixed: why it matters for agents

Without `stage_fixed = true`, a formatter's output changes the working tree
but doesn't end up in the commit — the agent retries, gets the same
failure, and loops forever. betterhook's `stage_fixed` snapshots the
unstaged file set before the job, diffs it after, and re-adds any newly-
modified files so the commit captures the fix.

<Tip>
  Always set `stage_fixed = true` on `fmt`-style jobs. It's the difference
  between a working self-correction loop and an infinite one.
</Tip>

## Loop termination

Cap the self-correction loop at one retry. If `fix → run` still fails,
the remaining errors need a real code change — hand them back to the
LLM, don't loop on `fix`.
