## v1.0.0 — 2026-04-11

The launch release. Adds the capability DAG scheduler, the
content-addressable hook cache, on-save speculative execution,
monorepo `[packages.<name>]` blocks with path filters, KDL alongside
TOML / YAML / JSON, persistent per-repo daemons via launchd / systemd,
twelve builtin linter wrappers with structured JSON diagnostics, and
the `betterhook doctor` health check. Single binary — `betterhookd`
is gone in favor of a hidden `betterhook serve` subcommand.

### Added

<AccordionGroup>
  <Accordion title="Capability DAG" icon="git-merge">
    - Schema gains `reads`, `writes`, `network`, `concurrent_safe` per job
    - Resolver builds a real DAG, runs every root in parallel, serializes only conflicting pairs
    - `priority = [...]` becomes a tiebreaker, not the only ordering signal
    - `betterhook status` and `betterhook explain` surface the resolved graph
  </Accordion>
  <Accordion title="Content-addressable cache" icon="database">
    - Keyed on `blake3(content) + blake3(tool_binary) + blake3(args)`
    - Tool binary hashing follows mise / nvm shims so upgrades naturally invalidate
    - Per-input mtime freshness gate so a stale prewarm never silently passes
    - `betterhook cache stats | clear | verify` for agent introspection
  </Accordion>
  <Accordion title="Speculative execution" icon="zap">
    - `notify`-backed file watcher with graceful degradation on NFS / sandboxed containers
    - Daemon prewarms `concurrent_safe` jobs on save, results land in the CA cache
    - Commit-time runner just hits the cache — golden path is zero subprocesses
  </Accordion>
  <Accordion title="Monorepo packages" icon="boxes">
    - `[packages.<name>]` config blocks with `path` filters
    - Per-package hook inheritance with override and skip semantics
    - Packages run in parallel; git-index writes serialized via shared mutex
  </Accordion>
  <Accordion title="Builtins + doctor + import" icon="package">
    - 12 builtins: rustfmt, clippy, prettier, eslint, ruff, black, gofmt, govet, biome, oxlint, shellcheck, gitleaks
    - Each with structured JSON diagnostics on the same NDJSON stream
    - `betterhook doctor` walks 7 health checks, exits non-zero on fail
    - `betterhook import --from <husky|lefthook|hk|pre-commit>` replaces the v0 `migrate` command
  </Accordion>
</AccordionGroup>

### Changed

- `betterhook migrate` is now a hidden alias for `betterhook import --from-format lefthook`
- `betterhookd` is no longer a separate binary — `betterhook serve` is the new entry point

## v0.0.1 — 2026-04-11

Initial scaffolding release. Built across 20 phases in the project plan
and ready for in-repo dogfooding.

### Added

<AccordionGroup>
  <Accordion title="Foundation" icon="layers">
    - Turborepo monorepo with `apps/betterhook` (library + `betterhookd` binary) and `apps/cli` (the `betterhook` CLI)
    - Cargo workspace on resolver v3, edition 2024, Rust 1.86 MSRV
    - Multi-format config parser supporting TOML, YAML, JSON, all lowering to one canonical AST via serde
    - `extends` inheritance with depth-first resolution, cross-format support, circular detection, and `betterhook.local.*` override layer
    - Async git worktree introspection via `tokio::process::Command` — never libgit2/gix, deliberately
    - Fileset templates (`{staged_files}`, `{push_files}`, `{all_files}`, `{files}`) with NUL-delimited parsing and `globset` filtering
  </Accordion>

  <Accordion title="Install model" icon="download">
    - Single byte-identical POSIX wrapper in the shared `<common-dir>/hooks/` dir
    - At runtime, wrapper uses `git rev-parse --show-toplevel` to identify the current worktree and exec's into `betterhook __dispatch`
    - SHA-256 manifest in `<common-dir>/betterhook/installed.json` so uninstall never touches user-modified hooks
    - `core.hooksPath` takeover/refuse semantics for coexistence with other hook tools
    - `__dispatch` subcommand resolves the per-worktree config and emits soft-miss exits for unconfigured worktrees
  </Accordion>

  <Accordion title="Execution engine" icon="play">
    - Sequential executor with line-streaming subprocess output via a central `OutputMultiplexer` actor
    - Parallel executor with a `tokio::sync::Semaphore` and priority-ordered spawn — directly fixes lefthook #846
    - Shared `Cancel` latch ensures in-flight run_commands SIGKILL their children on `fail_fast`
    - `kill_on_drop(true)` + explicit abort of stdout/stderr reader tasks on cancel or timeout
    - Untracked-file stash safety via unique-message `git stash` with verified pop — fixes lefthook #833
    - `stage_fixed` via before/after unstaged snapshots, with parallel jobs serialized through a shared git-index mutex
    - Per-job timeouts parsed by humantime, enforced in the select!, reported as exit code 124
    - `BETTERHOOK_SKIP` / `BETTERHOOK_ONLY` env vars and matching CLI flags
  </Accordion>

  <Accordion title="Agent surfaces" icon="bot">
    - NDJSON `--json` output with stable event kinds (`job_started`, `line`, `job_finished`, `job_skipped`, `summary`)
    - `betterhook status` prints a JSON snapshot of installed hooks, SHA integrity, resolved config, and worktree identity
    - End-to-end linked-worktree integration tests verify the headline property
  </Accordion>

  <Accordion title="Coordinator daemon" icon="server">
    - `betterhookd` opt-in daemon with length-prefixed bincode over a Unix socket
    - `LockKey → Arc<Semaphore>` registry with lazy creation and 60s idle linger
    - Per-connection `HashMap<token, HeldPermit>` so connection drop releases every held permit
    - `fs4` advisory flock fallback under `<common-dir>/betterhook/locks/<key>.lock`
    - Automatic `CARGO_TARGET_DIR=<worktree>/target` injection for `ToolPath::PerWorktree` on cargo
    - `BETTERHOOK_NO_LOCKS` / `--no-locks` escape hatch
  </Accordion>

  <Accordion title="CLI surfaces" icon="terminal">
    - `betterhook run <hook>` with `--dry-run`, `--json`, `--skip`, `--only`
    - `betterhook explain --hook <name> [--job]` returns a JSON snapshot of resolved jobs
    - `betterhook fix` runs every job's `fix = ...` variant — the agent self-correction surface
    - `betterhook migrate --from lefthook.yml` converts real lefthook configs and writes migration notes
  </Accordion>

  <Accordion title="Tooling" icon="wrench">
    - Criterion benches: `config_parse` and `output_multiplexer`
    - `xtask bench` driver with CI-gating exit codes
    - cargo-fuzz targets for the multi-format parser and the dispatch path detection
    - Fuzz crate excluded from the main workspace so stable-toolchain builds never need nightly
  </Accordion>

  <Accordion title="Documentation" icon="book">
    - Full README with install, config, agent surfaces, architecture, exit codes
    - `docs/protocol.md` — daemon IPC wire format
    - `man/betterhook.1` — subcommand reference
    - This Mintlify docs site under `apps/docs/`
  </Accordion>
</AccordionGroup>

### Known gaps

- Daemon client is flock-only in v0.0.1; socket-backed acquire path, spawn retry, and health handshake land next
- Stash safety covers the untracked-file case but not the stacked "unstaged delta under staged delta" case — a patch-based approach is planned for a later phase
- `xtask stress` and the nightly lefthook-compat diff suite are stubbed
- `remotes:` config inheritance (with SHA pinning) deferred to v0.3
- `LockfileGuarded` isolation (the prettier-on-pnpm-lock fix) deferred to v0.3
