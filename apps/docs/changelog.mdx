## v0.1.0 â€” 2026-04-29

First public release. The CLI, runner, daemon, cache, and the multi-format
config parser are all in place; the docs site, recipes, and packaging
scaffolds (Homebrew, npm, crates.io) ship with it.

### Added

<AccordionGroup>
  <Accordion title="Configuration" icon="settings">
    - Multi-format parser: TOML, YAML, KDL, and JSON, all lowering to a single canonical AST
    - `extends` inheritance with depth-first resolution, cross-format support, circular detection, and a `betterhook.local.*` override layer
    - Fileset templates `{staged_files}`, `{push_files}`, `{all_files}`, `{files}` with `globset` filtering
    - `[packages.<name>]` blocks for monorepos with path filters and per-package hook inheritance
    - Per-job capability declarations: `reads`, `writes`, `network`, `concurrent_safe`
  </Accordion>

  <Accordion title="Runner" icon="play">
    - Sequential and parallel executors with line-streaming subprocess output through a single `OutputMultiplexer` actor
    - Capability DAG scheduler â€” independent jobs run in parallel, conflicting pairs serialize automatically
    - Shared `Cancel` latch propagates `fail_fast` aborts to every in-flight child, including SIGKILL plus reader-task abort to avoid orphan-pipe hangs
    - Per-job timeouts via humantime, reported as exit code 124
    - `stage_fixed` via before/after unstaged snapshots, with parallel jobs serialized through a shared git-index mutex
    - Untracked-file stash safety with verified pop
  </Accordion>

  <Accordion title="Install model" icon="download">
    - Single byte-identical POSIX wrapper installed into the shared `<common-dir>/hooks/`
    - At runtime the wrapper resolves the active worktree via `git rev-parse --show-toplevel` and exec's into `betterhook __dispatch`
    - SHA-256 manifest in `<common-dir>/betterhook/installed.json` so uninstall never touches user-modified hooks
    - `core.hooksPath` takeover semantics for coexistence with husky / pre-commit
  </Accordion>

  <Accordion title="Coordinator daemon" icon="server">
    - Opt-in `betterhook serve` daemon with length-prefixed bincode over a Unix socket
    - `LockKey â†’ Arc<Semaphore>` registry with lazy creation and idle-exit lifecycle
    - Per-connection held-permit map so a connection drop releases every permit
    - `fs4` advisory flock fallback when the daemon is unavailable
    - `BETTERHOOK_NO_LOCKS` / `--no-locks` escape hatch
  </Accordion>

  <Accordion title="Content-addressable cache" icon="database">
    - Keyed on `blake3(content) + blake3(tool_binary) + blake3(args)`
    - Tool binary hashing follows mise / nvm shims so version bumps invalidate cleanly
    - Per-input mtime freshness gate prevents stale prewarms from passing silently
    - `betterhook cache stats | clear | verify` for introspection
  </Accordion>

  <Accordion title="Speculative execution" icon="zap">
    - `notify`-backed file watcher with graceful degradation on NFS and sandboxed containers
    - Daemon prewarms `concurrent_safe` jobs on save; results land in the cache so commit-time runs are cache hits
  </Accordion>

  <Accordion title="Builtins, doctor, import" icon="package">
    - 12 builtin wrappers: rustfmt, clippy, prettier, eslint, ruff, black, gofmt, govet, biome, oxlint, shellcheck, gitleaks
    - Structured diagnostics emitted on the same NDJSON stream as job output
    - `betterhook doctor` walks health checks for install, config, tools, cache, and watcher
    - `betterhook import --from <husky|lefthook|hk|pre-commit>` for migrating existing configs
    - `betterhook migrate` retained as a hidden alias for `import --from-format lefthook`
  </Accordion>

  <Accordion title="CLI surfaces" icon="terminal">
    - `betterhook run <hook>` with `--dry-run`, `--json`, `--skip`, `--only`
    - `betterhook explain --hook <name> [--job]` prints the resolved plan without executing
    - `betterhook fix` runs every job's `fix = ...` variant
    - `betterhook status` prints a JSON snapshot of installed hooks, SHA integrity, resolved config, and worktree identity
    - `betterhook completions <shell>` for bash, zsh, fish, elvish, powershell
    - NDJSON `--json` event stream with stable kinds (`job_started`, `line`, `job_finished`, `job_skipped`, `summary`)
    - `BETTERHOOK_SKIP` / `BETTERHOOK_ONLY` env vars and matching CLI flags
  </Accordion>

  <Accordion title="Docs, recipes, packaging" icon="book">
    - Mintlify docs site under `apps/docs/`
    - Drop-in `betterhook.toml` recipes in `recipes/` for common stacks
    - Homebrew formula and npm wrapper scaffolds in `packaging/`
    - Criterion benches and a cargo-fuzz harness for the multi-format parser and dispatch path
  </Accordion>
</AccordionGroup>
