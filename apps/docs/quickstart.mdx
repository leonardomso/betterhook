title: "Quickstart"
description: "Go from zero to a working worktree-aware pre-commit hook in 60 seconds."
icon: "rocket"
---

<Steps>
  <Step title="Install the binary">
    ```sh
    cargo install betterhook-cli
    # or, from a checkout:
    # cargo install --path apps/cli
    ```

    On macOS, a release tarball is also available on GitHub. Drop the binary into `/usr/local/bin`.
  </Step>

  <Step title="Scaffold a starter config">
    ```sh
    cd your-repo
    betterhook init
    ```

    Writes `betterhook.toml` at the repo root with two sample jobs
    (`cargo fmt --all -- --check` and `cargo clippy --workspace -- -D warnings`).
    Edit it for your stack.
  </Step>

  <Step title="Install the hook wrapper">
    ```sh
    betterhook install
    ```

    Writes a POSIX wrapper to `<git-common-dir>/hooks/pre-commit`. The wrapper
    is byte-identical for every worktree of this repo. At commit time it
    runs `git rev-parse --show-toplevel` and dispatches to whichever worktree
    is committing.

    <Note>
      If husky or pre-commit have claimed `core.hooksPath`, pass `--takeover` to replace them.
    </Note>
  </Step>

  <Step title="Verify">
    ```sh
    betterhook status | jq
    ```

    You should see:

    ```json
    {
      "betterhook_version": "0.1.0",
      "worktree": { "path": "...", "git_dir": "...", "common_dir": "..." },
      "installed": {
        "hooks": [{ "name": "pre-commit", "present": true, "matches": true }]
      },
      "config": { "hooks": [{ "name": "pre-commit", "jobs": ["fmt", "lint"] }] }
    }
    ```
  </Step>

  <Step title="Commit something">
    ```sh
    echo "hello" > hi.txt
    git add hi.txt
    git commit -m "first hook"
    ```

    Each job in `betterhook.toml` runs, output streams line by line, and the
    commit is blocked if anything fails.
  </Step>
</Steps>
