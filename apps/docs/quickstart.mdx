<Steps>
  <Step title="Install the binary">
    ```sh
    cargo install --path apps/cli
    # or, once published:
    # cargo install betterhook-cli
    ```

    On macOS you can also grab a release tarball from GitHub and drop the binary into `/usr/local/bin`.
  </Step>

  <Step title="Scaffold a starter config">
    ```sh
    cd your-repo
    betterhook init
    ```

    This writes a `betterhook.toml` at the repo root with two sample jobs
    (`cargo fmt --all -- --check` and `cargo clippy --workspace -- -D warnings`).
    Edit it to match your stack.
  </Step>

  <Step title="Install the hook wrapper">
    ```sh
    betterhook install
    ```

    This writes a tiny POSIX wrapper into `<git-common-dir>/hooks/pre-commit`.
    The wrapper is **byte-identical across every worktree of this repo** — at
    commit time it runs `git rev-parse --show-toplevel` and dispatches to
    whichever worktree is actually committing.

    <Note>
      If you already use husky or pre-commit and they've claimed `core.hooksPath`,
      pass `--takeover` to replace their setup.
    </Note>
  </Step>

  <Step title="Verify">
    ```sh
    betterhook status | jq
    ```

    You should see:

    ```json
    {
      "betterhook_version": "0.0.2",
      "worktree": { "path": "...", "git_dir": "...", "common_dir": "..." },
      "installed": {
        "hooks": [{ "name": "pre-commit", "present": true, "matches": true }]
      },
      "config": { "hooks": [{ "name": "pre-commit", "jobs": ["fmt", "lint"] }] }
    }
    ```
  </Step>

  <Step title="Commit something">
    ```sh
    echo "hello" > hi.txt
    git add hi.txt
    git commit -m "first hook"
    ```

    betterhook runs every job in `betterhook.toml`, streams output line by line,
    and blocks the commit if anything fails.
  </Step>
</Steps>
